# security-tutorial

[security-tutorial](https://github.com/ccss17/security-tutorial) 은 한동대학교 보안동아리 고스트 신입부원의 2주간의 컴퓨터 보안 학습 지도를 위하여 제가 만든 일종의 가이드입니다.

## 가이드의 특징

첫 1주간에는 리버싱을 공부하기 위하여 컴퓨터의 밑바닥부터 공부한 후 본격적인 리버싱을 학습하게 됩니다. 나머지 1주간에는 공부했던 리버싱을 기반으로 프로그램에 존재하는 취약점을 해킹해보는 시간을 갖게 됩니다.

이 가이드의 특징은 컴퓨터를 있게 한 수학적 모델을 이해해보고 컴퓨터의 구조 및 원리를 자세히 이해하게 하여 컴퓨터 자체에 대한 이해도를 심화시킨다는 것에 있습니다. 또한 32비트 시스템 뿐만 아니라 64비트 시스템에 대한 컴퓨터 구조와 원리도 이해시켜 줍니다.

이후 Rensselaer 대학에서 만든 http://security.cs.rpi.edu/courses/binexp-spring2015 라는 리버싱 문제를 풀어보면서 자연스럽게 리버싱을 익히고, 제가 직접 만든 [CTF](https://github.com/ccss17/bof) 문제들을 풀어보면서 자연스럽게 어떻게 취약점이 발생하고 해킹이 이루어질 수 있는지 이해하게 됩니다.

마지막으로 이러한 온실 속 화초같은 취약점에서 벗어나서 현실 세계에서 어떻게 취약점이 발생하는지 [Meltdown](../paper/meltdown/) 과 [Blueborne](../paper/blueborne/) 등을 살펴보게 됩니다. 놀라운 것은 이 2주간의 가이드를 통하여 이 2개의 현실세계의 취약점을 잘 이해할 수 있게 된다는 것입니다. 

또한 제가 만든 CTF 프로젝트 [bof](bof.md) 에서 언급하였듯이 (취약점 &rarr; 보안기법 &rarr; 취약점 &rarr; 보안기법 &rarr; ...) 의 싸이클을 볼 수 있는 통찰력을 갖게 합니다.

## Docker 

기존에 리버싱과 CTF 환경을 구축하기 위하여 이 과정을 서버 또는 VM 에서 진행하였지만, 여러가지 문제로 인하여 Docker 컨테이너로 완전히 이주시켰습니다.
