    Timecomplexity

    시간 복잡도(time complexity) : 입력에 대한 계산과정(알고리즘)이 시간을 얼마나 사용하는지 근사적으로 알려주는 도구이다.

    • 시간 복잡도의 \(O\)-표기법은 알고리즘 수행 시간의 상한을 알려준다.

    • \(\Omega\)-표기법은 알고리즘 수행 시간의 하한을 알려준다.

    • \(\Theta\)-표기법은 알고리즘 수행 시간을 정확하게 알려준다.

    시간 복잡도 \(\Omega\)-표기법 : \(k\)변수 함수 \(f\) 와 상수 \(c, n_1, n_1, \dots, n_k\) 에 대하여 크기가 \(i \in \{1, \dots,k\}\) 에 대하여 \(\mathbf{n} \geq n_i\) 인 모든 입력 \(\mathbf{n} \in \R ^{k}\) 에 대하여 계산과정(알고리즘)이 최소 \(cf(\mathbf{n})\) 번의 연산을 수행한다는 것을 \(\Omega (f(\mathbf{n}))\) 으로 나타내는 것이다.

    시간 복잡도 \(\Theta\)-표기법 : 시간복잡도가 \(O(f(n))\) 이면서 \(\Omega (f(n))\) 인 것을 \(\Theta (f(n))\) 로 나타내는 것이다.

    시간 복잡도 \(O\)-표기법 : \(k\)변수 함수 \(f\) 와 상수 \(c, n_1, n_1, \dots, n_k\) 에 대하여 크기가 \(i \in \{1, \dots,k\}\) 에 대하여 \(\mathbf{n} \geq n_i\) 인 모든 입력 \(\mathbf{n} \in \R ^{k}\) 에 대하여 계산과정(알고리즘)이 최대 \(cf(\mathbf{n})\) 번의 연산을 수행한다는 것을 \(O(f(\mathbf{n}))\) 으로 나타내는 것이다.

    • 시간 복잡도는 입력을 나타내는 \(n\) 변수 함수 \(f\) 에 대하여 \(O(f)\) 로 표기한다.

      Example

      입력이 정수 배열이면 \(n\) 은 배열의 크기를 나타낸다.

      입력이 문자열이면 \(n\) 은 문자열의 길이를 나타낸다.

    • 코드가 단일 명령어라면 시간 복잡도는 \(O(1)\) 이다.

      Example

      a++; b++;
c = a+b;

      위 코드의 시간 복잡도는 \(O(1)\) 이다.

    • 반복분의 시간 복잡도는 루프 안의 내용이 몇 번 수행되는지 근사적으로 알려준다. 일반적으로 루프가 \(n\)번씩 수행되고 루프가 \(k\) 번 중첩되어 있으면 시간 복잡도가 \(O(n ^{k})\) 이다.

      Example

      for (int i = 1; i <n ; i++)  { ... }

      위 코드의 시간 복잡도는 루프의 \(\dots\) 의 시간복잡도가 \(O(1)\) 이라는 가정 하에 루프가 \(n\) 번 수행되므로 \(O(n)\) 이다.

      Example

      for(int i = 1; i < n; i++)
    for (int j = 1; j < n; j++) {
        ...
    }

      위 코드의 시간 복잡도는 \(O(n ^{2})\) 이다.

    • 시간 복잡도가 코드의 수행 횟수를 정확하게 알려주지는 않는다. 함수의 차수만 남기고 상수 인자들은 무시되기 때문이다.

      Example

      for(int i = 1; i<= 3*n; i++) { ... }

for(int i = 1; i<= n+5; i++) { ... }

for(int i = 1; i<= n; i+=2) { ... }

      위 루프는 각각 \(3n, n+5, n/2\) 번 수행되지만 모두 시간 복잡도가 \(O(n)\) 이다.

      Example

      for (int i = 1; i<n; i++) 
    for (int j = 1; j <= i; j++) { 
        ... 
    }

      위 코드의 루프는 \(1 + 2 + \dots + n = \dfrac{1}{2}(n ^{2}+ n)\) 번 수행되므로 시간복잡도가 \(O(n ^{2})\) 이다.

    • 알고리즘이 여러 단계로 구성되어 있다면 전체 시간 복잡도는 각 단계의 시간 복잡도 중 가장 큰 것이 된다.

      Example

      for (int i = 1; i<=n; i++) { ... }
for (int i = 1; i<=n; i++) 
    for (int j = 1; j<=n; j++) { ... }
for (int i = 1; i<=n; i++) { ... }

      위 코드는 각각 시간 복잡도가 \(O(n), O(n ^{2}), O(n)\) 인데 전체 시간 복잡도는 \(O(n ^{2})\) 이다.

    • 시간 복잡도가 여러 인자의 영향을 받을 때도 있다.

      Example

      for (int i = 1; i<=n; i++) 
    for (int j = 1; j<=m; j++) { ... }

      위 코드의 시간복잡도는 \(O(nm)\) 이다.

    재귀 함수의 시간 복잡도 : 재귀 함수의 시간 복잡도는 함수가 몇 번 호출되는지, 그리고 각 호출 때의 시간 복잡도를 곱한 것이다.

    • 예시

      Example

      void f(int n){
    if(n == 1)return;
    f(n-1);
}

      \(f(n)\) 을 수행하면 함수 호출이 \(n\) 번 발생하고, 각 호출의 시간 복잡도는 \(O(1)\) 이다. 따라서 전체 시간 복잡도는 \(O(n)\) 이다.

      Example

      void g(int n) {
    if(n == 1) return;
    g(n-1);
    g(n-1);
}

      위 함수에 인자를 \(n\) 으로 주면 인자가 \(n-1\) 인 함수 호출이 \(2\) 번 일어나고, 인자가 \(n-2\) 인 함수 호출이 \(4\) 번 일어나고, 인자가 \(n-3\) 인 함수 호출이 \(8\) 번 일어난다. 즉 \(k \in \{0,1,\dots,n-1\}\) 에 대하여 인자가 \(n-k\) 인 함수 호출이 \(2 ^{k}\) 번 일어난다.

      따라서 함수 호출 횟수는 \(1 + 2 + 4 + \dots + 2 ^{n-1} = 2 ^{n} -1\) 인데, 각 함수 호출의 시간 복잡도는 \(O(1)\) 이므로 전체 시간 복잡도는 \(O(2 ^{n})\) 이다.

    시간 복잡도 종류 : 시간 복잡도에는 자주 사용되는 형태들이 있다.

    • 다음은 알고리즘이 빠른 순서대로 시간 복잡도 종류를 나열한 것이다.

    • \(O(1)\) : 상수 시간 알고리즘(Constant-time algorithm)

      수행 시간이 입력의 크기에 영향을 받지 않는다.

    • \(O(\log_{}n )\) : 로그 시간 알고리즘(Logarithmic algorithm)

      대체로 단계마다 입력의 크기가 절반씩 줄어든다. \(n\) 을 계속 \(2\) 로 나누어가며 \(1\) 이 되도록 하는데 필요한 단계 수는 \(\log_{2} n\) 이므로 이런 알고리즘의 수행 시간은 로그 시간이다.

    • \(O(\sqrt[]{n})\) : 제곱근 시간 알고리즘(Square root algorithm)

      제곱근은 \(\sqrt[]{n} = \dfrac{n}{\sqrt[]{n}}\) 이 성립하는데, 이로써 \(n\) 개의 원소를 각각 \(\sqrt[]{n}\) 개씩 이루어진 원소로 이루어진 그룹 \(O(\sqrt[]{n})\) 로 나눌 수 있다.

    • \(O(n)\) : 선형 시간 알고리즘(Linear algorithm)

      입력을 모두 쭉 살펴보는 과정을 상수 번 수행한다. 일반적으로 선형 시간이 가장 효율적인데, 이는 답을 구하기 위해 입력을 적어도 한 번은 쭉 살펴보기 때문이다.

    • \(O(n \log_{} n)\)

      입력을 정렬하는 과정의 시간 복잡도를 기술할 때 사용된다. 효율적인 정렬 알고리즘의 시간 복잡도가 이것이기 때문이다.

      또 연산을 수행할 때마다 \(O(\log_{} n)\) 시간이 걸리는 자료구조를 사용하는 알고리즘 등을 설명할 수 있다.

    • \(O(n ^{2})\) : 제곱 시간 알고리즘(Quadratic algorithm)

      보통 2중 루프를 표현할 때 사용한다. 입력 원소 두 개로 만들 수 있는 모든 조합을 한 번씩 살펴보는 시간이기도 하다.

    • \(O(n ^{3})\) : 세제곱 시간 알고리즘(Cubic algorithm)

      보통 3중 루프를 표현할 때 사용한다. 입력 원소 세 개로 만들 수 있는 모든 조합(triplet)을 한 번씩 살펴보는 시간이기도 하다.

    • \(O(2 ^{n})\)

      입력 원소로 만들 수 있는 모든 부분집합을 한 번씩 살펴보는 알고리즘의 시간 복잡도이다.

    • \(O(n!)\)

      입력 원소로 만들 수 있는 모든 순열을 한 번씩 살펴보는 알고리즘의 시간 복잡도이다.

    다항 시간 알고리즘(polynomial algorithm) : 시간 복잡도가 상수 \(k\) 에 대해 \(O(n ^{k})\) 를 넘지 않으면 다항 시간 알고리즘이라 한다.

    • 시간 복잡도의 종류에서 \(O(2 ^{n}), O(n!)\) 을 제외하고 모두 다항 시간이다.

    • 실제 상황에서 \(k\) 가 매우 작으며 시간 복잡도가 다항 시간이라는 것은 알고리즘이 매우 큰 입력도 처리할 수 있다는 말이다.

      • 그러나 중요한 문제 중에서 아직 다항 시간 알고리즘이 알려지지 않은 문제들, 즉 NP-하드 문제들이 많이 존재한다.
